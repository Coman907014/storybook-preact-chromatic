import { Meta } from '@storybook/addon-docs';


<Meta title="Dev Guidelines/Accessibility" />

<div className="row start-align col-lg-12 col-md-12 col-sm-12 pb-16">
  <h1 className="row start-align col-lg-12 col-md-12 col-sm-12">Accessibility - The Basics</h1>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Keyboard Access and Visual Focus:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    All content and functionality should be available through a keyboard. Keyboard focus should be visible and should follow a logical order through the page elements.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Page Title and Language:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    The page title should adequately and briefly describe the content of the page and should be different from other pages on the website. Primary language of every page and language changes should be indicated.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Headings:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    The page should have at least one heading. It should be marked up correctly as a heading and the heading hierarchy should be meaningful.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Images:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Every image should include alt in the markup. If an image conveys information useful, then it needs alternative text. If an image is just decorative, then it should have null alt (alt="").
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Contrast, Color, Appearance:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    The page should have a contrast ratio of at least 4.5:1 for normal-size text. Color alone should not be used to convey information.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Resizing, Magnification, Responsiveness:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Users should be able to enlarge web content. The design should be responsive.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Links, Buttons, Controls, Navigation, Skip Link:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Links, buttons and controls should be recognizable and coded correctly. Clear and consistent navigation options should be provided. A skip link that allows users to skip to content should be available.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Forms, Labels, Errors:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Forms should be keyboard accessible. Form controls should have labels. Required fields and other instructions should be clearly indicated. Error handing should be clear and simple.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Tables and Lists:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Tables should have HTML markup that indicates header cells and data cells and defines their relationship. List elements should be used for list content.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Multimedia (Audio and Video), Animation, Moving content:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Captions/subtitles, transcript, and/or audio description should be provided appropriately for audio and video media. Keyboard access for media player controls, animations, carousels and other moving content should be available.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Structure, Meaning, Reading Order, Global Code:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Content should be well-structured to allow efficient navigation and processing. Mark-up should be used for headings, lists, tables, etc. to better structure content. Reading order should be reflected in the code order.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-8">
  <h2 className="row start-align col-lg-12 col-md-12 col-sm-12">
    Touch:
  </h2>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    Touch interactions should be supported. When phone is in scope, phone interactions should be supported.
  </div>
</div>

<div className="row start-align col-lg-12 col-md-12 col-sm-12 py-16 font-italic">
  <div className="row start-align col-lg-12 col-md-12 col-sm-12">
    For in-depth information about this topic, please reference <a style={{ paddingLeft: '4px' }} href="https://www.w3.org/WAI/standards-guidelines/" target="_blank"> W3C Accessibility Standards </a>
  </div>
</div>
