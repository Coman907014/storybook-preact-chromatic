<div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8" id="touchTargets">

<div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8">
  <div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-2">
    Touch targets are the parts of the screen that respond to user input. They extend beyond the visual bounds of an element.â€¨For example, an icon may appear to be 24 x 24 px, but the padding surrounding it comprises the full 48 x 48 px touch target.
  </div>
  <div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8 pb-8">
    <h3>Minimum Touch Target Size</h3>
    Touch targets should be at least 48 x 48 px. A touch target of this size results in a physical size of about 9mm, regardless of screen size. For smaller components like icons and checkboxes this can be built up using padding to reach the target size.
  </div>
  <div className="row center-align col-lg-12 col-md-12 col-sm-12 pt-8">
    <img src={SpacingTouchTargets} />
  </div>
  <div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8">
    <h3>Space Between Touch Targets</h3>
    To avoid mis-hits on touch devices and enforce a cursor change when using a mouse, touch targets should be separated by a minimum of 8px.
  </div>
  <div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8">
    <h3>Exceptions</h3>
    Inline elements such as links do not need to meet the minimum touch target size as this can cause links to overlap if used within a paragraph.
  </div>

  <div className="row start-align col-lg-12 col-md-12 col-sm-12 pt-8">
    <h3>Pointer Targets</h3>
    Pointer targets are similar to touch targets, but apply to the use of motion-tracking pointer devices such as a mouse or a stylus. 
  </div>
</div>
